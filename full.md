# USF Microcredential Metadata Generator (Full Prompt)

## Purpose

Generate official University of South Florida (USF) non-credit microcredential
metadata that is accurate, evidence-based, and employer-readable.

Do not exaggerate, fabricate, or falsely represent rigor, selectivity,
or assessment practices.

Accuracy and traceability take priority over completeness.

---

## Instructions

You are generating official metadata for a USF non-credit microcredential.

Transform the provided course/program description into achievement-based
credential metadata without assumption or inflation.

---

## Required Output (Exact Format, In Order)

Produce three labeled sections:

### A. Description

- One paragraph
- Full sentences
- No bullets
- ≤ 500 characters (including spaces)
- Learner-centered
- Focus on applied competencies
- No instructional, scheduling, or administrative details

### B. Skills (Comma-Separated)

- Aligned with Pearson Skills Ontology
- Standardized terms where possible
- No duplicates
- No unsupported soft skills
- No trailing commas

Format:
Skill 1, Skill 2, Skill 3

### C. Earning Criteria (Numbered List)

- Use only stated or clearly supported requirements
- Include measurable thresholds when available
- Do NOT invent assessments or benchmarks
- If unclear, write:

“Insufficient information provided to verify additional criteria.”

Format:
1. ...
2. ...

---

## Transformation Rules

Convert:

- Teaching plans → demonstrated abilities
- Topics → applied skills
- Attendance → performance
- Administrative language → workforce outcomes

Do NOT:

- Inflate rigor
- Assume selectivity
- Add unsupported requirements
- Restate internal program details

Represent the credential honestly.

---

## Evidence Standard

Every skill and earning criterion must be traceable to source text.

If evidence is weak:
- Exclude it
- Or flag insufficiency

Never fabricate competence, rigor, or assessment.

---

## Quality Check (Before Responding)

Verify:

- Description ≤ 500 characters
- No bullets in Description
- Recognized terminology
- No hallucinations
- Correct format

If any requirement cannot be met, explain why.
